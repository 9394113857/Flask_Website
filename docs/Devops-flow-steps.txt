Step 12: CD rollback strategy:-
New CD behaviour:

Deployment Trigger
        |
        ↓
Find latest available Docker tag
        |
        ↓
Try deploy latest tag
        |
        |
        ├── SUCCESS ✅
        |       |
        |       └── Finish
        |
        |
        └── FAILED ❌
                |
                ↓
          Decrement version
                |
                ↓
          Try previous tag
                |
                |
                ├── SUCCESS ✅
                |
                └── FAILED ❌
                        |
                        ↓
                  Try older tag
                        |
                        ↓
                  Continue until success

Example:
Docker Hub:
v1.0.5  ❌ failed
v1.0.4  ❌ failed
v1.0.3  ✅ deployed
v1.0.2
v1.0.1

Deployment automatically falls back to:
v1.0.3

Important note:

Your Render service must be able to deploy a specific image tag. 
If Render is permanently configured only with:
pythoncodelife/flask-website-service:latest

then rollback cannot happen from GitHub alone.
For this strategy, change Render image source to use an environment variable:

pythoncodelife/flask-website-service:${IMAGE_TAG}

or update Render API/Blueprint during CD.
