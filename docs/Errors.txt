Ask your wife:

What is Python trying to import?

calculator_bp

from

Then ask

Does calculator_controller.py actually contain a variable named calculator_bp?

Probably No.

That's why Python says:

I looked inside calculator_controller.py, but I couldn't find calculator_bp.

In __init__.py you probably have:
from app.controllers.calculator_controller import calculator_bp

So Python expects this inside calculator_controller.py:

calculator_bp = Blueprint(...)

If it's missing, you'll get exactly this error.

There are only 3 possible reasons
Reason 1 (Most Common)

Your file is empty.

Example:

# calculator_controller.py

Nothing inside.

❌ Error.

Reason 2

Blueprint name is different.

Example

calc_bp = Blueprint(...)

But importing

calculator_bp

❌ Error.

Names must match.

Reason 3

Blueprint wasn't created.

Example

from flask import Blueprint

but forgot

calculator_bp = Blueprint(...)
Quick Fix

Create a placeholder controller first.

app/controllers/calculator_controller.py

from flask import Blueprint, render_template

calculator_bp = Blueprint("calculator", __name__)


@calculator_bp.route("/calculator")
def calculator():
    return render_template("calculator.html")

Then create an empty HTML.

templates/calculator.html

{% extends "base.html" %}

{% block title %}
Calculator
{% endblock %}

{% block content %}
<h1>Calculator Page</h1>
{% endblock %}

All of those controller files must exist and define the expected Blueprint variables.

My recommendation (Professional way)

Before adding any logic, create all controller placeholders.

controllers/

bmi_controller.py

flames_controller.py

guess_controller.py

from flask import Blueprint, render_template

<name>_bp = Blueprint("<name>", __name__)


@<name>_bp.route("/<route>")
def <function_name>():
    return render_template("<page>.html")

Then create the matching HTML file.

Now the project starts successfully.
